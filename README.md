# DevCleaner

DevCleaner is a Rust CLI tool that removes unused ES module imports from JavaScript/TypeScript files.

It scans a project recursively, detects imported identifiers that are never referenced, and rewrites import declarations.

## What it supports

- File types: `.js`, `.jsx`, `.ts`, `.tsx`
- Recursive scanning from a root directory
- Multi-folder projects (for example `Front/` + `Back/` under a single root)
- Skips common heavy/system folders:
  - `node_modules`
  - `.git`
  - `target`
  - `dist`

## Output modes

DevCleaner has two write modes:

1. **In-place mode (default)**
   - Rewrites files directly in your project.

2. **Dist mode**
   - Writes cleaned files to `dist/` preserving the relative folder structure.
   - Enabled when:
     - You pass `--dist`, or
     - The root directory name is `sample` (test convenience mode).

## Installation

- From crates.io:
  - `cargo install dev-cleaner-cli`
- From local source (this repository):
  - `cargo install --path .`

After installation, the command is:

- `dev-cleaner-cli`

## Available commands

CLI syntax:

- `dev-cleaner-cli [path] [--dist]`

Examples:

- Scan current directory (in-place):
  - `dev-cleaner-cli`
- Scan a specific directory:
  - `dev-cleaner-cli /path/to/project`
- Force `dist/` output:
  - `dev-cleaner-cli /path/to/project --dist`
- Run against included samples:
  - `dev-cleaner-cli sample`

## Usage

From the repository root:

- Scan current directory (in-place):
  - `cargo run --release`

- Scan a specific directory (in-place by default):
  - `cargo run --release -- /path/to/project`

- Force `dist/` output:
  - `cargo run --release -- /path/to/project --dist`

- Run against included samples (auto `dist` mode):
  - `cargo run --release -- sample`

## Example project layout

If your project is:

```
/Project
  /Front
  /Back
```

Run DevCleaner at `/Project` and it will process both `Front` and `Back` recursively.

## How it works (high level)

1. Parse each module with `swc_ecma_parser`.
2. Collect imported local names from `import` declarations.
3. Collect identifier usages outside import declarations.
4. Compute unused imports with set difference.
5. Remove unused import specifiers.
6. Drop now-empty import declarations.
7. Emit code with `swc_ecma_codegen`.

## Current limitations

- Focused on classic import specifiers (`named`, `default`, `namespace`).
- Does not currently provide a "report-only" or "dry-run" mode.
- Formatting is generated by SWC emitter and may differ from your formatter.

## Planned improvements

- **Dry run mode**
  - Analyze files without writing changes.
- **Removed-items report**
  - Show which imports/specifiers were removed per file.
- **Check mode for CI**
  - Exit with non-zero status when removable imports are detected, without rewriting files.

## Benchmark (Rust vs ESLint)

Internal benchmark results (not published yet), using the same sample set:

| Tool | Median | p95 | Avg | Approx. files/s |
|---|---:|---:|---:|---:|
| DevCleaner (Rust) | 76.57 ms | 85.76 ms | 77.34 ms | 104.47 |
| ESLint (`no-unused-vars`) | 191.29 ms | 213.88 ms | 192.57 ms | 41.82 |

Estimated speedup (median): **2.50x** in favor of DevCleaner.

> Note: benchmark numbers can vary depending on hardware, OS, file count, and project structure.
